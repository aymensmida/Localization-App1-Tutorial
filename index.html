Laravel Localization

Working with translation files

Localization is the process of adapting a website or application to a particular language, culture, and region. Laravel is a popular PHP web framework that provides built-in support for localizing applications.

So, in Laravel, just like in many other frameworks, you will store translations for different languages in separate files. There are two ways to organize Laravel translation files:

  • An old approach which involves storing your files under the following path: resources/lang/{en,fr,ru}/{myfile.php}.

  • A new approach of having resources/lang/{fr.json, ru.json} files.

Our example will be with the second approach

I-Install Laravel 8

Open terminal further run below command to manifest a new laravel project, ignore this step if project is already installed

composer create-project laravel/laravel:^8.0 laravel-Localization

II- Translation files

Now create resources/lang/en.json resources/lang/fr.json resources/lang/ar.json 

touch resources/lang/en.json resources/lang/fr.json resources/lang/ar.json
en.json 
                        {
                            "salutation":"welcome",
                            "object":"Translation Test",
                            "result":"Translation completed successfully"
                        }
fr.json 
                        {
                            "salutation":"Bienvenue",
                            "object":"Test de traduction",
                            "result":"Traduction terminée avec succès"

III-Create Routes

Open the routes/web.php file and define the following route:

                        <?php

                        use Illuminate\Support\Facades\Route;
                         
                            /*
                        |--------------------------------------------------------------------------
                        | Web Routes
                        |--------------------------------------------------------------------------
                        
                        */
                        
                        Route::get('/', function () {
                            return view('welcome');
                        });
                        Route::get('{locale}', function ($locale) {
                            app()->setLocale($locale);
                            session()->put('locale', $locale);
                        
                            return redirect()->back();
                        })->name('switch_language');

IV-Middleware 

php artisan make:middleware Localization
                        <?php


                        namespace App\Http\Middleware;

                        use Closure;
                        use Illuminate\Http\Request;
                        use Session;
                        use Illuminate\Support\Facades\App;
                        class Localization
                    {
                      public function handle(Request $request, Closure $next)
                       {
                        if (Session::has('locale')) {
                        App::setLocale(Session::get('locale'));
                        }
                       return $next($request);
                        }
                     }

Since we need this operation to be run on every request, we also need to add it to the default middleware stack at app/http/Kernel.php for the web middleware group
